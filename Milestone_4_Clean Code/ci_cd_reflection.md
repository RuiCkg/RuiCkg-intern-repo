# CI/CD Reflection - Rui Chosa

## Tasks

### Research what CI/CD is and why it’s used in software development

CI/CD stands for Continuous Integration and Continuous Delivery or Continuous Deployment.

CI helps developers automatically test and check code whenever new changes are pushed. This helps catch problems early.

CD extends this process by automatically preparing or deploying the application after the checks pass.

### What is the purpose of CI/CD

The purpose of CI/CD is to automate the software development workflow and improve code quality. It helps teams catch errors earlier, reduce manual work, and make development faster and more reliable.

### How does automating style checks improve project quality

Automating style checks improves project quality by making documentation more consistent and easier to read. Markdown linting catches formatting issues, and spell checks catch writing mistakes before they are merged.

### What are some challenges with enforcing checks in CI/CD

Some challenges include false positives from spell checkers, strict formatting rules, and dependency issues during setup. It can also take extra time to fix existing files before the checks pass.

### How do CI/CD pipelines differ between small projects and large teams

In small projects, CI/CD pipelines are usually simple and may only include linting, spell checks, and basic tests. In larger teams, they often include automated tests, security scans, builds, deployment stages, and approval steps.
