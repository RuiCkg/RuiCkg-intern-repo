# Roles in an Agile Team - Rui Chosa
## Research & Learn
### What are the key roles in an Agile team, and what does each role do?
- Developers: Build the product, write code, fix bugs, and implement features.
- QA (Quality Assurance): Test the product to find bugs and ensure quality.
- Product Manager (PM): Difines product vision, sets priorities, and manages the backlog.
- Agile Project Manager (Agile PM): Facilitates the process, removes blockers, and ensures the team follows Agile practices.
- Other supporting roles (e.g., stakeholders, UX, customer support)
    - Stakeholders: Provide requirements and feedback.
    - UX Designers: Design user experience and interface.
    - Customer Support: Share user feedback and report common issues.

### How do these roles collaborate in Kanban vs. Scrum?
In Kanban, collaboration is continuous and flexible, while in Scrum, work is done in fixed sprints with clearly defined roles and structured meetings.
